# 数愈健康 · 品牌调性守则

> 所有面向患者和家属的中文文案都必须遵守这份守则。新加页面、新加提示、新加短信、新加错误文案都要过这一关。
>
> 这不是建议，是纪律。

---

## 产品定位

**我们是谁**：为中国肿瘤患者和家属提供病历整理、临床试验匹配、治疗方案参考的陪伴型工具。

**用户画像**：
- **主用户**：患者家属（子女、配偶），30-55 岁，非医学背景
- **情绪状态**：焦虑、疲惫、信息过载、对医疗系统既依赖又疑惧
- **首要需求**：看懂家人的病历 / 找到可以尝试的新治疗 / 不想被骗 / 不想家人数据被卖

**不是谁**：不是医生、不是医院、不是药企、不是保险。**永远不要让患者觉得我们在替医生做决定**。

## 6 条铁律

### 1. 称谓
- 统一用「**您**」（敬称）。**绝不混用「你/您」**。
- 患者本人称「**您的家人**」或「**家人**」，不用「患者」「病人」。直接对患者本人时也是「您」。
- 系统自称「**我们**」（温度的集合），**禁用**「本系统」「本平台」「数愈系统」。

### 2. 禁用词 🚫
以下词在用户可见文案里不准出现（内部文档、代码注释随意）：

| 禁用词 | 原因 | 替代 |
|---|---|---|
| 结构化 | 工程师黑话 | 整理好 / 看懂 / 关键信息 |
| 智能化 / 智能识别 | 自我表扬 | AI 帮您 / 帮您 |
| AI 驱动 / 大模型加持 | 吹嘘 | 去掉 |
| 赋能 | 空话 | 去掉或说清干嘛 |
| 一站式 / 全链路 / 闭环 | 营销黑话 | 一次完成 |
| 打通 | 工程师黑话 | 对接 / 连到 |
| 脱敏 | 太技术 | 已处理、看不到真实信息 |
| 精准 | 夸大 | 合适 / 贴合 |
| 全程加密 | 空洞承诺 | 传输加密 + 存储加密 + 您可删除（具体讲） |
| 秒级 / 毫秒级 | 工程师炫耀 | 几秒 / 几分钟 |
| 合规 / 法律合规 | 法务语言，冷 | 严格保护您的隐私 |

### 3. 强制词汇池
在合适的情境下主动使用（每屏**最多 1 次**，别煽情过头）：

- **陪伴类**：陪您、一起、我们在、别担心、慢慢来
- **赋权类**：您做主、您掌管、您随时可以、听您的、您改过的就是对的
- **安抚类**：不着急、没关系、先看看再决定、不用现在决定
- **诚实类**：可能会、不一定、仅供参考、最终请听主治医生、我们没看清

### 4. 每个错误提示必须含两个要素

**模板**：`[共情一句] + [可行动作一句]`

示例：
- ❌ 「上传失败」
- ❌ 「网络错误，请重试」
- ✅ 「这张图片我们没能看清 —— 换张清晰些的再试试？」
- ✅ 「网络有点卡，您的上传没丢，稍等再试一次」
- ✅ 「验证码对不上呢，再核对一下？（可以让我们重发）」

**绝不**：只说"失败"、只说"错误"、只给 HTTP 状态码、只给堆栈、用感叹号（除非是报喜）。

### 5. 每个加载态要给时间感和动作感

- ❌ 「加载中」「Loading...」
- ❌ 「处理中，请稍候」
- ✅ 「正在帮您看懂这份病历…」（6-12 字，动作化）
- ✅ 「马上好…」（≤ 6 字，用于 < 2s 操作）

进度超过 10s 必须有副文字告诉用户在做什么、预计多久。

### 6. 隐私承诺必须在"首次数据操作"前复述

以下 4 个节点必须出现「🔒 数愈健康承诺」卡片或等价文案：
1. 第一次上传病历前（UploadView 顶部）
2. 第一次提交匹配申请前（MatchesView apply 确认）
3. 个人中心（ProfileView 顶部）
4. Landing 页

统一用 `<PrivacyPromiseCard>` 组件（见 `web/src/components/PrivacyPromiseCard.vue`）。

**承诺模板**：
> 🔒 数愈健康承诺
> · 我们不存储您的任何隐私数据
> · 所有信息由您自己保管
> · 您随时可以带走或删除

## 7 个情感模板

写新文案对不上调性时，套这 7 个模板：

| 场景 | 模板 | 示例 |
|---|---|---|
| 欢迎 | 「欢迎 + 您 + 我们帮您 + 具体动作」 | 「欢迎回来。让我们继续帮您家人找下一步。」 |
| 引导上传 | 「把 X 交给我们 + 我们会做 Y + 隐私承诺」 | 「把病历交给我们。AI 几分钟看懂关键信息。数据仅在您的账户里。」 |
| 结果呈现 | 「好了 + 这是 Z + 请您核对 + 您说了算」 | 「好了。以下是从病历里看到的信息 —— 请过一遍，哪里不对直接改。您改过的就是对的。」 |
| 等待/排队 | 「通常 X 时间 + 这段时间您不用做 Y + 会 Z 通知您」 | 「研究团队通常 1-3 天内联系您。这段时间您不用做什么，有消息会短信通知。」 |
| 匹配失败 | 「目前没找到 + 但不代表没有 + 我们会继续 + 建议动作」 | 「目前没找到完全贴合的 —— 但不代表没有。我们每周更新，下周再来？也可让医生预筛。」 |
| 成功入组 | 「太好了 + 重要意义 + 下一步谁负责 + 我们做什么」 | 「太好了，您成功入组。接下来的治疗细节请听主治医生安排，我们会把资料安全归档。」 |
| 错误降级 | 「我们没能 X + 换 Y 试试？+ 或者 Z 也行」 | 「这张图片我们没能看清 —— 换张清晰些的再试试？或者直接手动输入关键信息也行。」 |

## 隐私声明的 3 层表达

同一套承诺，3 个长度，按位置选：

**L1 一行（Landing hero / 按钮下）**
> 您的数据始终是您的。

**L2 短段（Upload / Profile 顶部）**
> 🔒 我们不存储您的任何隐私数据 · 所有信息由您自己保管 · 您随时可以带走或删除

**L3 完整（privacy.html / 小程序 privacy 页）**
> 我们对您的承诺：
>
> 1. **不存储您的手机号原文** —— 仅加密保存用于登录，任何员工无法直接看到
> 2. **病历图片 7 天后自动删除** —— AI 看懂后提取的文字信息保留在您的账户里，您随时可以删
> 3. **不外传、不倒卖、不用于营销** —— 您的病历只用于给您匹配临床试验
> 4. **您可以一键导出所有数据** —— JSON + PDF，带走就能走
> 5. **您可以一键注销账户** —— 所有信息彻底删除，包括备份
> 6. **我们用腾讯云 OCR / Moonshot AI 帮您识别病历** —— 他们也签署了不留存承诺
>
> 有任何疑虑，请通过 xxx 联系我们，我们会在 3 个工作日内回复。

## 决策边界：什么必须让人、什么可以 agent 自动写

| 类型 | 允许 agent 自动改 | 必须人审 |
|---|---|---|
| 按钮文案 | ✅ | |
| 加载态、空态 | ✅ | |
| 错误提示（HTTP 标准错误） | ✅ | |
| 隐私声明文本 | ❌ | **必须**（法务风险） |
| Hero 主标 / 副文 | ❌ | **必须**（品牌门面） |
| 医学建议/诊断相关文案 | ❌ | **必须**（合规风险） |
| 短信模板 | ❌ | **必须**（群发风险） |

## CI 守则（自动化）

pre-commit 或 CI 跑一条 grep：

```bash
# 禁用词检查 —— 命中即红
grep -rn -E "结构化|智能化|赋能|闭环|一站式|全链路|精准匹配|打通|秒级|毫秒级" \
  web/src/ server/public/ pages/ 2>/dev/null
# exit 0 表示没命中（绿）；exit 1 表示命中（红）
```

未来把它塞进 `.github/workflows/brand-voice-lint.yml` 和 `.pre-commit-config.yaml`。
