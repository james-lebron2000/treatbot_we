# TreatBot 用户使用手册

> 版本：v19 | 更新日期：2026-04-02
> 访问地址：https://inseq.top/treatbot/

TreatBot 是一个临床试验智能匹配平台，帮助癌症患者快速找到适合的临床试验，同时为 CRO/申办方提供患者线索管理工具。

## 患者端使用指南

### 1. 登录

**地址：** https://inseq.top/treatbot/login

1. 输入 11 位手机号
2. 点击 **发送验证码**（60 秒内只能发送一次）
3. 输入收到的 4 位验证码
4. 点击 **登录**

登录后自动跳转到上传页面。

---

### 2. 上传病历

**位置：** 底部导航栏 → 上传

支持上传以下类型的医疗文件：
- 病理报告（图片或 PDF）
- 出院小结
- 基因检测报告
- 影像检查报告

**操作步骤：**

1. 点击选择文件，支持一次上传多个文件
2. 系统会自动进行 AI 识别和解析（约 30-120 秒）
3. 解析完成后，系统提取出以下关键信息：
   - 诊断（如"非小细胞肺癌"）
   - 分期（如"IV期"）
   - 基因突变（如"EGFR L858R"）
   - ECOG 体能评分
   - 治疗史
4. 如果有缺失的字段，可以手动补充
5. 确认无误后点击 **查看匹配结果**

**提示：**
- 上传多份报告时，系统会自动合并信息
- 图片请拍摄清晰、光线充足
- PDF 文件不超过 30MB

---

### 3. 查看匹配结果

**位置：** 底部导航栏 → 匹配

系统会根据您的病历信息，从 496 个临床试验中智能匹配，按匹配度从高到低排列。

**匹配分数含义：**
- 80% 以上（绿色）：高度匹配，建议重点关注
- 60-79%（橙色）：部分匹配，建议了解详情
- 60% 以下（灰色）：匹配度较低

**筛选功能：**
- 按试验阶段筛选（I期 / II期 / III期）
- 按城市筛选
- 按招募状态筛选（招募中 / 已关闭）

每条匹配结果展示：
- 试验名称和匹配度分数
- 适应症、机构、阶段
- 匹配理由（绿色标签，如"诊断匹配""基因突变匹配"）

点击 **查看详情** 了解试验完整信息。

---

### 4. 试验详情

点击匹配结果中的 **查看详情**，可以看到：

- **基本信息**：试验名称、匹配度、适应症、申办方
- **匹配依据**：系统为何认为您可能符合该试验
- **研究介绍**：试验的目的和方案说明
- **入排标准**：详细的入选条件和排除条件
- **研究中心**：可以参与该试验的医院列表
- **报名所需资料**：需要准备的文件清单
- **患者补助**：参与试验可能获得的补助信息
- **联系方式**：研究中心的联系电话

确认有意愿后，点击页面底部的 **申请报名** 按钮。

---

### 5. 申请管理

**位置：** 底部导航栏 → 申请

查看所有已提交的试验申请及其状态：

| 状态 | 含义 |
|------|------|
| 待联系（黄色） | 申请已提交，等待研究机构联系 |
| 已联系（蓝色） | 研究机构已与您取得联系 |
| 已入组（绿色） | 已通过筛查，成功入组 |
| 未通过（红色） | 筛查后不符合条件 |
| 已取消（灰色） | 您已主动取消 |

**取消申请：** 在"待联系"或"已联系"状态下，可以点击 **取消申请** 按钮撤回。

---

### 6. 我的

**位置：** 底部导航栏 → 我的

查看个人信息和使用统计：
- 绑定的手机号
- 昵称
- 已上传病历数
- 已匹配试验数

点击 **退出登录** 可安全退出。

## CRO 端使用指南

CRO（合同研究组织）/ 申办方工作人员使用独立的登录入口管理患者入组。

### 1. 登录

**地址：** https://inseq.top/treatbot/cro/login

1. 输入管理员分配的公司邮箱
2. 输入密码
3. 点击 **登录**

如忘记密码，请联系平台管理员重置。

---

### 2. 入组看板

登录后进入入组看板，这是 CRO 的核心工作界面。

#### 选择试验

页面顶部的下拉框中列出了分配给您公司的所有试验，每个试验后面显示当前申请人数。选择试验后加载该试验的入组看板。

#### 看板状态

看板按 5 个状态分为不同的标签页：

| 状态 | 说明 | 操作建议 |
|------|------|----------|
| 待筛查 | 患者刚提交申请，尚未处理 | 查看诊断信息，判断是否初步符合 |
| 已联系 | 已与患者取得联系 | 记录沟通结果，安排筛查 |
| 已入组 | 通过筛查，成功入组 | 完成入组 |
| 已排除 | 筛查后不符合条件 | 记录排除原因 |
| 已取消 | 患者主动取消 | 无需操作 |

#### 患者卡片信息

每张患者卡片展示：
- **姓名**和**手机号**（中间四位脱敏显示，如 138****1193）
- **诊断标签**：诊断、分期、基因突变、治疗线数、PD-L1 表达
- **申请时间**

#### 操作

**更改状态：** 点击卡片右上角的状态下拉框，选择新状态即可。卡片会自动移动到对应标签页。

**添加备注：** 在卡片底部的输入框中输入沟通记录，点击 **记录** 保存。卡片会显示最近 3 条备注。

**导出数据：** 点击页面底部的 **导出线索 (CSV)** 按钮，下载当前试验的所有申请数据表格，包含：
- 患者信息（昵称、手机号）
- 诊断摘要（诊断、分期、基因突变、治疗线、PD-L1）
- 申请状态
- 沟通备注

---

### 3. 退出

点击页面右上角的 **退出** 按钮安全退出 CRO 系统。

## 管理员操作指南

管理员通过患者端登录（手机号 + 验证码），在"我的"页面可以看到管理入口。

### 进入管理后台

1. 用管理员手机号登录患者端
2. 点击底部导航栏 **我的**
3. 点击 **管理后台** 按钮

### 管理后台功能

#### 概览

显示平台核心指标：
- 总用户数（今日新增）
- 总病历数（今日新增）
- 总申请数（今日新增）
- 总试验数

#### 申请管理

查看所有患者提交的试验申请，可以：
- 通过状态下拉框更新申请状态
- 查看患者诊断信息和申请时间
- 分页浏览

#### 用户列表

查看所有注册用户：
- 昵称、手机号
- 上传病历数
- 提交申请数
- 注册时间

#### CRO 管理

**创建 CRO 账号：**
1. 填写公司名称、联系人姓名
2. 填写登录邮箱和密码
3. 填写负责的试验 ID（多个用逗号分隔）
4. 点击 **创建账号**

**管理 CRO 账号：**
- 查看已创建的 CRO 公司列表
- 点击 **禁用** 暂停 CRO 访问权限
- 点击 **启用** 恢复权限

#### 数据导出

- **导出用户数据**：下载所有用户信息 CSV
- **导出病历数据**：下载所有病历及匹配结果 CSV
- **导出全部申请**：下载所有试验申请 CSV
- **按试验导出**：输入试验编码，导出该试验的申请线索 CSV

### CRO 入组看板（管理员版）

在"我的"页面点击 **CRO 入组看板**，管理员可以看到所有试验的入组数据（不受试验分配限制）。

## 常见问题

### 患者端

**Q: 上传病历后一直在解析怎么办？**
A: AI 解析通常需要 30-120 秒。如超过 2 分钟仍未完成，页面会显示重试按钮，点击即可重新解析。

**Q: 匹配结果准确吗？**
A: 匹配结果基于 AI 分析您的病历信息与试验入排标准的契合度，仅供参考。最终是否符合入组条件，需由研究机构的医生判断。

**Q: 申请后多久会有人联系我？**
A: 通常 3 个工作日内研究机构会与您取得联系。

**Q: 我的病历信息安全吗？**
A: 您的所有医疗数据都经过加密传输和存储，仅用于临床试验匹配，不会用于其他用途。

**Q: 可以同时申请多个试验吗？**
A: 可以。您可以根据匹配结果申请多个试验，由研究机构分别联系您。

### CRO 端

**Q: 如何获取 CRO 账号？**
A: 请联系 TreatBot 平台管理员，提供公司名称、联系人和邮箱，管理员会为您创建账号并分配试验。

**Q: 为什么看不到某个试验？**
A: CRO 账号只能看到管理员分配给您的试验。如需增加试验权限，请联系管理员。

**Q: 导出的 CSV 文件乱码怎么办？**
A: CSV 文件使用 UTF-8 编码。用 Excel 打开时，请选择"数据" → "从文本/CSV导入"，编码选择 UTF-8。
